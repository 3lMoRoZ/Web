Ecco i tre standard fondamentali che sono alla base del Web:

HTML (HyperText Markup Language):

HTML è il linguaggio di markup utilizzato per creare la struttura di base delle pagine web. Consiste in un insieme di elementi (o "tag") che vengono utilizzati per definire la struttura logica dei contenuti, come testo, immagini, link ipertestuali, tabelle, paragrafi e altro ancora.
Gli elementi HTML sono organizzati in una struttura ad albero, con un elemento <html> radice che contiene <head> (dove vengono inserite informazioni come il titolo della pagina e metadati) e <body> (dove si trovano i contenuti effettivi visualizzati sulla pagina).


URL (Uniform Resource Locator): 

È l'indirizzo che identifica una risorsa specifica su internet. Consiste in una sequenza di caratteri che definisce il percorso per accedere a una pagina web, un'immagine, un file o qualsiasi altra risorsa online.
Un URL è composto da diversi elementi che specificano la posizione e il tipo di risorsa a cui si sta accedendo. Ecco i principali componenti di un URL:
1) Schema: Lo schema indica il protocollo utilizzato per accedere alla risorsa. Ad esempio, "http://" o "https://" per le pagine web, "ftp://" per il trasferimento di file, "mailto://" per indirizzi email, "file://" per file locali, ecc.

2) Host: Il nome dell'host indica il server su cui risiede la risorsa desiderata. Può essere un dominio come "www.example.com" o un indirizzo IP numerico.

3) Percorso: Il percorso specifica la posizione della risorsa all'interno del server. Ad esempio, "/pagina.html" per una pagina web specifica, "/immagini/foto.jpg" per un'immagine, ecc.

4) Query string (opzionale): La query string è una serie di parametri che seguono il percorso e sono separati da un punto interrogativo "?" in un URL. Questi parametri forniscono informazioni aggiuntive alla risorsa richiesta. Ad esempio, in un URL come "https://www.example.com/pagina.html?parametro1=valore1&parametro2=valore2", la query string contiene i parametri "parametro1" e "parametro2" con i rispettivi valori.

5) Fragment (opzionale): Il frammento, indicato da un simbolo "#" seguito da un identificatore, indica una sezione specifica all'interno della risorsa. È spesso utilizzato per indicare una posizione specifica in una pagina web. Ad esempio, in "https://www.example.com/pagina.html#sezione2", "#sezione2" indica la sezione 2 della pagina HTML.


HTTP (Hypertext Transfer Protocol):

HTTP è il protocollo di comunicazione utilizzato per il trasferimento di informazioni tra un client (come un browser web) e un server (dove risiedono le risorse web). Funziona su un modello richiesta-risposta: il client invia una richiesta HTTP al server per ottenere risorse come pagine web, immagini o file e il server risponde con i dati richiesti.
L'HTTP funziona tipicamente sul modello di richiesta "GET" per ottenere dati e "POST" per inviare dati a un server, ma esistono anche altri metodi HTTP come "PUT", "DELETE", "PATCH" e altri usati per operazioni specifiche.

Questi tre standard lavorano insieme per consentire la creazione, la presentazione e la distribuzione di contenuti web su Internet, costituendo le fondamenta dell'esperienza di navigazione che viviamo quotidianamente durante la fruizione di pagine web.
